### 2. 創作エピソードを書かない
NG例（過去に失敗した典型）：
- ❌「俺、最初の5社は全部落ちた」
- ❌「最初の3回は真顔でいきなり自己紹介始めてた」

### 3. 一般論を入れない
就活界で誰でも言ってる話（ChatGPTでも書ける内容）はNG。
DBから具体を引っ張る。

NG例：
- ❌「業界を3つに絞れ」（DB出典なしの一般論）
- ❌「なぜ？を5回繰り返せ」（DB出典なし）
- ❌「ガクチカを2個仕込め（学内・学外）」（DB出典なし）

### 4. 書く前に必ずDB検索
手順：
1. スライドのテーマ決定
2. 先にDBクエリで該当投稿を探す
3. 原文をコピーしてから、キャラの口調に加工する
4. 該当なしなら**そのスライドを書かない**

### 5. 冗長な実績フレーズを入れない
- タイトルで「通過率100%」を出したら、本文で繰り返さない
- 「その後10社全部通った 通過率100%」みたいな本文内の数字もりもりNG

## 🎯 タイトル作成ルール

### 鉄則：読み手のメリットが明確か
「〇〇した話」「〇〇の体験」は**絶対NG**（読み手に何のメリットもない）。

**公式**：
```
【権威】＋【具体的にもらえるもの】
```

- 権威：「大手〇社内定の先輩」「元面接官」「通過率〇%」
- もらえるもの：テンプレ／神回答／模範解答／質問リスト／正解の選択肢／フレーズ集／チェックリスト

### TikTok DB実在フック型（これだけ使う）

| 型 | 実例 | 使い所 |
|---|---|---|
| **本当に知っておいてよかった** | 「本当に知っておいてよかった / 大手8社内定の先輩からもらった / 落ちることがなくなるES最強テンプレ集」 | 汎用テーマ全般 |
| **真似するだけで受かった** | 「真似するだけで受かった / 業界の中でなんでウチ？ / 大手6社から内定もらった解答例」 | 即実践型テンプレ配布 |
| **これさえ見ておけば〇〇無双間違いなし** | 「これさえ見ておけば面接は無双間違いなし / 面接通過率90%の私が教える頻出質問集」 | チェックリスト・網羅系 |
| **内定率がグーンとあがった** | 「内定率がグーンとあがった / 大手8社内定の先輩が使っていたぶっ刺さる逆質問」 | SHUKA定型、効果訴求 |
| **対策せずに〇〇すると爆死する** | 「内定者が教えてくれた、対策せずに面接すると爆死する / 想定質問12選」 | 恐怖訴求、危機感喚起 |
| **落ちることがなくなる** | 「落ちることがなくなるES最強テンプレ集」 | テンプレ配布 |
| **マジでもっと早く知りたかった** | 「マジでもっと早く知りたかった / 25卒が就活を終えて実感した / ほとんどの人が誤解してる就活の嘘話」 | 逆張り系 |
| **おい〇卒 〇月は絶対これやれ** | 「おい26卒 3月は絶対これやれ」 | 時期訴求・命令 |
| **質問引用型** | 「大手5社内定持ちの姉が使っていた / 「なぜその大学？」/ 面接官が納得する模範解答」 | 特定質問への回答 |

### NG表現リスト（AI臭い・TikTokにない）
- ❌「〜した話」だけで終わるタイトル
- ❌「今日はそのやり方を置いてく」
- ❌「〜って評価される」
- ❌「別物になった」「爆上がり」
- ❌「地味だけどクソ大事な〇〇戦略」
- ❌「〇〇って不思議じゃない？」
- ❌「考えてる〇〇だな」
- ❌「4ブロック埋めるだけで『考えてるESだな』って評価される」
- ❌ 広告コピー調のサブタイトル

### 数字の使い方
- **数字は結果ベース**：「通過率100%」「大手5社内定」「サマー16社獲得」
- **行動量ベースはNG**：「サマー40社出した」（出すだけなら誰でもできる＝権威にならない）
- **100% > 99% > 90%**：数字のインパクトは非線形。キリのいい100%が最強
- **90%は避ける**：ただ高いだけで止まらない

## 📐 スライド構成ルール

### 枚数
- **標準：10枚**
- **許容：8〜10枚**（2枚までの減少はOK）
- **禁止：7枚以下**（中身が薄くなる）

### TikTok実データとの関係（参考）
- TikTok実データでは5〜6枚が最頻（47%）
- 10枚は実データで3.6%の少数派
- ただし10枚には10枚の情報量的メリットあり
- 現行は10枚で継続、必要に応じて8枚まで許容

### 1スライドあたり文字数
- TikTok実データ中央値：**145字**
- 許容範囲：**100〜180字**
- 1スライドに情報詰め込みすぎ禁止（1テーマ1スライド）

### 基本構成
```
スライド1：タイトル（フック）
スライド2：問題提起 or 結論先出し
スライド3〜9：本体（型・テンプレ・チェックリスト・実例）
スライド10：CTA（フォロー促進＋ツール誘導）
```

### 情報密度の目安（10枚の場合）
- 総字数：1800〜2200字
- 数字：10〜20個
- 箇条書き記号（✅❌→①）：15〜30個
- 引用フレーズ：3〜7個

# 第3部：失敗から学んだこと（次回以降の禁忌）

## よくやった失敗ランキング

### 1位：創作で枠を埋める
- 「10枚埋めなきゃ」と思って一般論やでっち上げを追加した
- 対策：**DBにないなら枠を減らす**（ただし8枚までに留める）

### 2位：「出した数」を実績と勘違いする
- 「サマー40社出した」は行動量であって結果じゃない
- 対策：**通過率・内定数・結果ベースの数字だけ使う**

### 3位：タイトルに読み手メリットを入れない
- 「〇〇した話」で終わるタイトルは「で、何？」になる
- 対策：**何がもらえるか（テンプレ／リスト／フレーズ）を明示**

### 4位：AI臭い言い回しを入れる
- 「今日はそのやり方を置いてく」「〜って評価される」
- 対策：**TikTok DBにない言い回しは使わない**

### 5位：キャラ設定にこだわりすぎる
- 「大手5社内定」に固執して表現が固まる
- 対策：**年齢・性別・学歴・口調だけ固定、数字は毎回変える**

### 6位：タイトルが長すぎる
- 具体情報を3段重ねして60字超え
- 対策：**40〜55字に収める（TikTok実測 25%ile 41字 / 中央値 48字 / 75%ile 62字）**

### 7位：ヒット投稿の情報密度を舐める
- 「10枚に薄く情報を広げる」のではなく「箇条書き連発＋数字ゴリ押し＋テンプレ全文」が正解
- 対策：**1スライドに4〜8項目の箇条書き、数字を遠慮なく入れる**

# 第5部：もえキャラ作成時のFB集（2026/04/13-14）

2本目のキャラ（もえ：27卒女・日東駒専・萌OL参照）で4本作成する過程で得られた追加FB。次回以降の新キャラ作成・台本追加時は必ずこの章を読むこと。

## 5-1. キャラ設計ルール

### 語尾のカタカナ化は気持ち悪い【絶対NG】
- ❌ 「〜ダヨ」「〜ヨ」「〜ネェ」「〜だヨ」「〜ッ」（強調のカタカナ）
- DBの萌OL原文にはこれらが多用されているが、**原文をそのままコピーすると気持ち悪い**
- ⭕ 「〜かも」「〜だよね」「〜しといた方がいい」「体言止め」で締める
- DB原文の口調を参考にしつつ、現代の読み手に違和感ないレベルに調整する

### 権威性は「ネガ単体」だと逆効果【重要】
- ❌ 「ただの私文だった27卒の私が」単体 → ただの不安要素になる
- ⭕ 「ただの私文だけど東大早慶相手にGD無敗だった」= ネガ＋逆転ポジのセット
- 逆転ストーリーとして成立させるには**必ず「でも〇〇」の後半が必要**

### 権威性に多様性を持たせる【重要】
- 「大手内定数」ばかり使うのはNG
- テーマに合った実績を選ぶ：
  - 面接対策テーマ → サマーインターン11社通った（参加数）
  - ESテーマ → ES通過率9割
  - スケジュールテーマ → 後輩に内定させた（教える立場）
  - 4月準備テーマ → インターン参加数
- **同じキャラでも台本ごとに実績数字を変えて良い**（キャラ設定を固めすぎない）

### 絵文字は1台本1〜2個まで
- 多用すると台本感・AI感が出る
- 🥺💯👏🤍✨を使うなら、CTA付近かフック部分にだけ

---

## 5-2. タイトル設計ルール（追加分）

### 3行全体が文として繋がること【必須】
- ❌ `サマーインターン11社通った27卒の私が / インターン面接を突破する会話術`
  → 「私が〜する会話術」で動詞連体形はOKだが、過去形と現在形が混在して不自然
- ⭕ `サマーインターン11社通った27卒の私が / 実際に使ってた面接会話術`
  → 「私が実際に使ってた〇〇」で主語と述語が自然に繋がる
- **体言止めで終わる場合も、2行目「〜の私が」と3行目の動詞連体形が過去形で揃っているか確認**

### 2行目（権威性行）は20字前後
- 既存ヒット: 「面接通過率100%だった27卒の俺が使ってた」＝21字
- 20字超えると読みづらい。25字以上は長すぎ
- 長くなる時は「使ってた」を3行目に移動する or 学歴情報を削る

### タイトル変更したらスライド2も必ず整合性チェック
- タイトル変更でスライド2の導入部分と矛盾することが多い
- 例: タイトル「40戦39勝」→「11社通った」に変更した時、スライド2の「70回以上練習して40戦39勝」も併せて修正必要
- **タイトル変更後はスライド2を必ず読み直す**

### 穴埋め系フックは「テンプレ/フォーマット」系テーマで強力
- 台本F「穴埋めするだけで落ちない/ガクチカ最強フォーマット」
- 台本K「このテンプレに穴埋めするだけ/サマーES志望動機」
- 「穴埋め」単体では弱いので「〜するだけで落ちない」等のベネフィットとセット
- ただし「1分で作れる」等の時短ベネフィットも並行訴求できる

### 矢印は「→→」が正解（下矢印ではない）
- TikTokは**横スワイプ**カルーセル = 右矢印が自然
- 下矢印（↓↓）は縦スクロール型（Instagram風）の名残り
- DBヒット投稿の実測: ↓↓＝0件、→＝20件、→→＝1件
- **必ず右矢印を使う**

---

## 5-3. 本文表現ルール

### 視覚依存表現は禁止【画像スライドなので見えない】
- ❌ 「赤字だけ企業ごとに変える」（何が赤字か画像からは分からない）
- ❌ 「🔴🔵🟢の3要素」（絵文字マーカーが何を指すか不明）
- ❌ 「左の図」「太字or図」「この部分」
- ⭕ 具体名で書く: 「企業名と強みの部分だけ変える」「粘り強さ・リーダー経験・自己成長欲の3要素」

### OCRエラー語は削除する
- 萌OL原文「マリンとかストップウォッチ持った面接官」の「マリン」は意味不明（OCRエラー疑い）
- DB原文だからといってそのまま使わず、**意味が通らない単語は削除**
- 判断基準: 読み手に意味が通じるかどうか

### ジャーゴン禁止【新卒向けには通じない】
- ❌ 「粒度を揃える」「＋α」「最適解」「30業界/90職種」
- これらはコンサル/IT業界で使うワード
- 日東駒専〜MARCH層の28卒には通じない
- ⭕ 「レベルを揃える」「意見の細かさを整理する」「焦らず自分のペースで逆算」「メガバンク/商社/IT/メーカーなど主要業界」

### 抽象動詞を具体化する
- ❌ 「刺さる」「爆上がり」「別物になる」「反応が変わる」「一気に差がつく」「印象変わる」
- ⭕ 面接官の具体的な反応・行動に置き換える：
  - 「刺さる」→「『よく調べてるね』と反応が返ってくる」
  - 「印象変わる」→「『結論から話せる子だ』と印象が残る」
  - 「差がつく」→「HPしか見てない他の就活生と差がつく」
- **数値化できる場合は数値化**（ただし創作数字はNG）

### 指示詞の宙吊り禁止
- ❌ 「ここがズレてたら」「こんな感じで」「それだけで」
- ⭕ 直前の内容を具体的に言い直す

## 5-4. スライド構成ルール

### スライド10は4ブロック構成【統一】
```
ブロック1: 問題提起（〇〇しても△△できない）
ブロック2: 深堀り/具体的な困りごと
ブロック3: 解決（ウカルテ就活＋機能説明）
ブロック4: CTA引き（検索してみて＋特典はこれだけじゃない →→）
```
- 5ブロック以上にすると目線が散る
- 各ブロック2〜4行、合計100文字前後が目安

### スライド10→11のブリッジ必須
- スライド10を「検索してみて」で完結させるとスライド11を読まない
- 末尾に「特典はこれだけじゃない →→」を追加して誘導

### スライド10は台本ごと個別、スライド11は共通プレゼント訴求
- スライド10：台本テーマと問題提起を結びつける（個別化）
- スライド11：全台本共通のプレゼント訴求（統一感・ブランディング）

### 各スライド冒頭に繰り返し見出しを入れない
- ❌ スライド3,4,5,6全部に「28卒が今やるべき月別スケジュール」を入れる
- ⭕ 各スライドには【4月】【5月】等の差分のみ

### 「月別スケジュール」等の階段型コンテンツは4ヶ月以内
- 1台本で5ヶ月以上並べると目線がダレる
- 必要なら「前半/後半」で分けるか、重要な月だけ詳細版で他は圧縮
